# Constitution Update Checklist v7.1.0 (Simplified)

When amending The HexTrackr Development Constitution (`/memory/constitution.md`), ensure all core principles and rights remain clear and enforceable.

## Version & Structure
- [ ] Version number updated correctly (currently 7.1.0)
- [ ] Previous version referenced if applicable
- [ ] Simplified structure maintained (Core Principles + Bill of Rights)
- [ ] Clear, concise language used throughout

---

## Core Principles Verification

### 1. Docker-First
- [ ] All application code runs in Docker containers
- [ ] Direct host execution of node/npm/npx prohibited
- [ ] Docker commands specified where applicable
- [ ] Port mappings clear (8989 → 8080)

### 2. Test-First (TDD/SDD)
- [ ] No production code without failing test first
- [ ] Test verification requirement before merge
- [ ] TDD/SDD methodology clearly stated
- [ ] Test-first principle is absolute

### 3. Context Accuracy
- [ ] Context gathering required before work begins
- [ ] Context sources specified (Memento, branch, framework docs)
- [ ] Accuracy confirmation requirement stated
- [ ] No work without verified context

### 4. Memory Governance
- [ ] Significant sessions recorded to Memento
- [ ] Summary/outcome storage requirement
- [ ] Future recall capability ensured
- [ ] Memory preservation prioritized

### 5. Framework Documentation
- [ ] Context7 usage for authoritative docs
- [ ] No stale/unverified references allowed
- [ ] Documentation storage requirement
- [ ] Framework accuracy mandated

### 6. Spec-Kit Workflow
- [ ] /specify → /plan → /tasks workflow required
- [ ] No implementation without specification
- [ ] Workflow order enforced
- [ ] Spec-driven development clear

### 7. Branch Discipline
- [ ] Feature/copilot branches only
- [ ] Direct main commits forbidden
- [ ] Branch strategy clear
- [ ] Safe branching enforced

---

## Bill of Rights Verification

### Non-Negotiables
- [ ] Right to Context – No code without full context
- [ ] Right to Testing – No untested code in project
- [ ] Right to Memory – All sessions preserved in Memento
- [ ] Right to Rollback – All operations reversible
- [ ] Right to Review – Significant changes require approval

### Rights Enforcement
- [ ] Each right clearly stated
- [ ] Enforcement mechanism implied
- [ ] No contradictions between rights
- [ ] Rights support core principles

---

## Practical Validation

### Enforceability
- [ ] All principles actionable by Claude/agents
- [ ] No ambiguous requirements
- [ ] Clear SHALL/SHALL NOT language
- [ ] Measurable compliance possible

### Consistency
- [ ] No conflicting principles
- [ ] Rights align with principles
- [ ] Unified development philosophy
- [ ] Clear priority when principles intersect

### Simplicity
- [ ] Unnecessary complexity removed
- [ ] Essential behaviors retained
- [ ] Easy to remember and follow
- [ ] Quick reference possible

---

## Amendment Process

### When Making Changes
- [ ] Identify which principle/right affected
- [ ] Ensure change maintains simplicity
- [ ] Verify no new contradictions introduced
- [ ] Update version number appropriately
- [ ] Document rationale in Memento

### Impact Assessment
- [ ] Change doesn't complicate constitution
- [ ] Enforcement remains practical
- [ ] Other principles unaffected
- [ ] Bill of Rights preserved

---

## Final Sign-off
- [ ] All core principles verified
- [ ] Bill of Rights intact
- [ ] Version updated
- [ ] Checklist matches constitution
- [ ] User approval obtained

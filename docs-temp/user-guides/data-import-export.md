# Importing and Exporting Data

This guide explains how to import and export data in HexTrackr.

## Importing Data

HexTrackr supports importing data from CSV files. This is the primary way to populate the application with vulnerability and ticket data.

### Importing Vulnerabilities

1. Navigate to the **Vulnerability Management** dashboard.
2. Click the **Import** button.
3. Select the **Scan Date** for the vulnerability data.
4. Choose the CSV file to import.

### Importing Tickets

1. Navigate to the **Tickets Management** page.
2. Open the **Settings** modal (cog icon in the header).
3. Go to the **Data Management** tab.
4. Click the **Import Tickets** button and select your CSV file.

## Exporting Data

You can export data from HexTrackr in various formats.

### Exporting from the UI

- **Tickets Page**: Use the `CSV`, `Excel`, `JSON`, `PDF`, and `HTML` buttons to export the currently filtered list of tickets.
- **Vulnerability Dashboard**: Use the **Export** button to export the current view (Table, Devices, or Vulnerabilities) as a CSV file.

### Exporting from the Settings Modal

The **Data Management** tab in the **Settings** modal provides more advanced export options:

- **Export Tickets**: Exports all tickets to a CSV file.
- **Export Vulnerabilities**: Exports all vulnerabilities to a CSV file.
- **Export All Data**: Exports all tickets and vulnerabilities to a single combined CSV file.

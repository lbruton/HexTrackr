# Contribution Guidelines

We welcome contributions to HexTrackr! Please follow these guidelines to ensure a smooth process.

## Development Workflow

1. **Create a new branch**: Create a new branch for your feature or bug fix.
2. **Make your changes**: Write your code and tests.
3. **Run the linter**: Run `npm run lint:all` to check for any issues.
4. **Run the tests**: Make sure all tests pass.
5. **Create a pull request**: Push your changes to your fork and create a pull request.

## Code Style

- Follow the rules defined in the `.eslintrc.js` file.
- Use `npm run lint` to check for issues.

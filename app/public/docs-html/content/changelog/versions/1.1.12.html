Version 1.1.12 - Light Mode Visual Depth & Theme Consistency

Release Status: ✅ Released
Release Date: 2025-11-04
Parent Issues: UI/UX Improvements

Overview

Restored Material Design elevation principles to light mode with blue-gray surface hierarchy, fixed missing CSS files across tickets.html and documentation portal, and unified search bar presentation with card-based design. This release brings light mode visual depth on par with dark mode, creating proper contrast and "pop" for elevated UI elements.

[1.1.12]

Changed

Light Mode Surface Hierarchy Redesign - 2025-11-04

Description: Implemented multi-layer surface hierarchy in light mode using blue-gray color palette, replacing flat white-on-white design with proper Material Design elevation principles.

Problem Solved:
Light mode had lost all visual depth - white cards on white background created a flat, undifferentiated interface where elements blended together. Dark mode had proper contrast and "pop" but light mode looked washed out and unprofessional.

Design Changes:

Color Hierarchy (darker foundation → lighter elevated elements):

  • Page Background: #d0dae6 - Darker blue-gray foundation
  • Card Headers: #e8eef5 - Lighter blue-gray for elevation
  • Card Bodies: #ffffff - Pure white for maximum contrast
  • Secondary Backgrounds: #c8d4e0 - Even deeper blue-gray for base surfaces

CSS Variables Updated (/app/public/styles/css-variables.css):

/* Light Mode Surfaces */
--hextrackr-bg-primary: #d0dae6;      /* Page background */
--hextrackr-bg-secondary: #c8d4e0;    /* Deeper backgrounds */
--hextrackr-bg-tertiary: #e8eef5;     /* Elevated elements */
--hextrackr-card-header-bg: #e8eef5;  /* Card headers */
--hextrackr-card-body-bg: #ffffff;    /* Card bodies */

Enhanced Depth Elements (/app/public/styles/shared/light-theme.css):

  • Navbar: Added subtle box-shadow and border for header distinction
  • Stats cards: Gradient background with hover effects (translateY + box-shadow)
  • Proper surface elevation hierarchy across all card components

Files Modified:

  • app/public/styles/css-variables.css (lines 15-25, 95-100)
  • app/public/styles/shared/light-theme.css (lines 58-88)
  • app/public/styles/shared/cards.css (chart-container, workspace-container backgrounds)

User Experience:

  • ✅ Cards now "pop" from the background with clear visual hierarchy
  • ✅ Light mode matches dark mode quality and professional appearance
  • ✅ Improved readability through proper contrast ratios
  • ✅ Consistent elevation across all UI surfaces

Notes

Design Philosophy

Material Design Elevation: Light mode now follows Material Design principles with proper surface elevation hierarchy. Darker foundation layers create depth, allowing lighter elevated elements (cards, modals) to "float" above with clear visual distinction.

Color Psychology: Blue-gray palette provides:

  • Professional, enterprise-appropriate aesthetic
  • Reduced eye strain compared to pure white backgrounds
  • Clear visual hierarchy without overwhelming contrast
  • Consistent with modern SaaS application design patterns

User Experience Improvements

  • Light Mode Users: Major quality of life upgrade for users who prefer light mode - now matches dark mode's professional appearance and visual depth
  • Cross-Page Consistency: Unified theming across vulnerabilities, tickets, and documentation portal eliminates visual dissonance when navigating between sections
  • Card-Based Design: Consistent use of white cards on colored backgrounds creates predictable, learnable UI patterns

Performance Impact

  • Zero performance impact - CSS-only changes
  • No JavaScript modifications required
  • Theme variables enable instant theme switching

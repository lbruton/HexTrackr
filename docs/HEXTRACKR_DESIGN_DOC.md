# HexTrackr Master Design Documentation

## Executive Summary

**HexTrackr** is a unified vulnerability and ticket management platform designed to solve the real-world challenges of network administrators managing thousands of devices across multiple states and ticketing systems.

## User Story & Problem Statement

### The Challenge

As a network administrator managing nearly 2,000 devices across 9 states, I was drowning in spreadsheets and notepad files while coordinating vulnerability remediation across multiple teams and ticketing systems.

The core problem: Our organization requires tickets in **multiple systems** for vulnerability resolution:

- **ServiceNow**: For change tracking and network operations technician assignments
- **Hexagon**: For field operations technician dispatch and on-site reboots
- **Tenable**: Weekly CSV exports requiring manual processing for VPR trend analysis

This fragmentation led to:

- ğŸ• Hours of manual data entry per week
- ğŸ“Š Inconsistent reporting across systems
- ğŸ”„ Duplicate work and potential errors
- ğŸ“ˆ Difficulty tracking vulnerability trends over time

### The Solution

HexTrackr consolidates these workflows into a single, purpose-built platform that:

- Generates tickets with proper formatting for both ServiceNow and Hexagon
- Imports and deduplicates vulnerability data from multiple scanners
- Tracks VPR trends with visual dashboards
- Provides one-click bundle downloads with all supporting documentation
- Maintains comprehensive documentation for team knowledge sharing

## Deployment & Operations

### Docker Deployment

```bash
# Development Setup
docker-compose up -d              # Start services
curl http://localhost:8989/health # Verify running

# Production Deployment
docker build -t hextrackr:latest .
docker run -d \
  -p 8989:8080 \
  -v ./data:/app/data \
  --name hextrackr \
  hextrackr:latest
```

### Maintenance Operations

```bash
# Database Backup
sqlite3 data/hextrackr.db ".backup data/backup/hextrackr-$(date +%Y%m%d).db"

# Documentation Update
npm run docs:generate     # Regenerate HTML from markdown
npm run docs:analyze      # Update architecture documentation

# Code Quality
npm run lint:all         # Run all linters
npm run test:e2e         # Run Playwright tests
```
